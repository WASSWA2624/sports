# 001-scope

## Goal
Lock the app to Flashscore-style public web parity as defined in `app-write-up.md`.

## Build
1. Freeze the north star as one-to-one parity with the public Flashscore web experience observed on March 23, 2026.
2. Confirm the product is a live scores and sports news platform, not a marketplace, creator tool, or community app.
3. Lock core parity surfaces: shell, scores board, competition pages, match center, team pages, news, favorites, search, odds, and ad-slot foundation.
4. Lock the technical stack: Next.js App Router, JavaScript, Prisma, MySQL, Styled Components, Redux Toolkit, and i18n.
5. Publish and approve the parity sign-off document in `docs/mvp-scope.md`.

## Done When
- Scope explicitly targets Flashscore-style parity.
- Removed features are clearly out of scope.
- Team has a single approved source of truth for all later phases.
