# 011-pay

## Goal
Enable paid slips and purchase lifecycle.

## Build
1. Integrate payment provider for one-time slip purchase.
2. Implement order/payment/invoice records and webhook handling.
3. Unlock paid slip content only after verified payment.
4. Implement refund/dispute-ready transaction states.
5. Add creator payout request flow with step-up auth.
6. Add fraud and anomaly checks (velocity, duplicate purchase, mismatch flags).

## Done When
- Purchase flow is reliable end-to-end.
- Access control is payment-backed.
- Payout and finance audit trails are complete.
